section ‹Transformation of State Abstraction (ToSA)›

text ‹This is a reasoning procedure for transformations of abstraction of the whole computation
  state, which we name ∗‹Transformation of State Abstraction (ToSA)›.
  Specifications must be given in MTF.
  The procedure can recognize items specifying identical objects and
    invoke the reasoning for transforming the object from the source abstraction to the target
    abstraction.›

text ‹Priority Convention:
▪ 4000: Termination
▪ 3200: Very Safe Normalization
▪ 3150: Assigning Zeros
▪ 3000: Normalization
▪ 2800: Disjunction in source part; Default normalization in source part
▪ 2600: Disjunction in target part; Default normalization in target part
        Divergence happens here!
        Existentially quantified variables are fixed here!
▪ 2100: Padding void holes after the last item. Rules capturing the whole items including
        the last item in the ‹∗›-sequence should have priority higher than this.
▪ 2000: Step-by-step searching
▪ ≤ 1999: Rules for searching specific object like value, variable, etc.
▪ 800:  Disjunction in target part
▪ ≤ 80: Rules for general searching. This feature is disabled in view shift
          because most of the global-state-level components are configured
          with properly search rules so the general search is not required.
›
